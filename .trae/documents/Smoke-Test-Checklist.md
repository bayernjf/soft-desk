# SoftDesk 冒烟测试清单

> 用途:每次发布或重大改动前,快速验证核心功能是否可用。
> 维护方式:随开发进程持续追加。新增功能时,请在对应模块下补充用例;新增模块时,追加一个章节。
> 勾选方式:`[ ]` 未测 / `[x]` 通过 / `[!]` 失败(失败请在备注列写现象)。

## 测试环境说明
- 启动命令:`npm run dev`(Vite + Electron,默认端口 5173,被占用时自动顺延到 5174 等)。
- **软件扫描 / 真实启动 / 窗口拖拽与最大化** 依赖 Electron 的 `window.softdesk` 桥接,必须在 **Electron 窗口**中测试。
- 在**浏览器**中预览时,store 会自动 fallback 到 mock 数据(`MOCK_SOFTWARE`),启动软件不会真正打开应用,窗口控制能力不可用。
